# Where to Go Next

You now have a working understanding of OpenVox: the Puppet language, the
resource abstraction layer, modules, Hiera, and a `git`-based deployment
workflow with `r10k`. Here is where to go to keep learning.

## Documentation

* [OpenVox documentation](https://docs.openvoxproject.org/) — the full reference
* [OpenVox Quickstart Guide](https://voxpupuli.org/openvox/quickstart/)
* [Installing OpenVox](https://voxpupuli.org/openvox/install/)
* [The Puppet language reference](https://docs.openvoxproject.org/openvox/latest/lang_summary.html)
* [Resource type reference](https://docs.openvoxproject.org/openvox/latest/type.html)

## Tools

* [Puppet Forge](https://forge.puppet.com/) — find and share modules
* [r10k](https://github.com/puppetlabs/r10k) — environment and module deployment
* [OpenVox DevKit](https://docs.openvoxproject.org/ecosystem/latest/devkit/) — create, validate, and test modules
* [OpenFact](https://docs.openvoxproject.org/openfact/latest/index.html) — the fact-collection library
* [OpenBolt](https://docs.openvoxproject.org/openbolt/latest/) — task and plan orchestration

## Community

* [OpenVox project on GitHub](https://github.com/OpenVoxProject)
* [Vox Pupuli](https://voxpupuli.org/) — the community that maintains OpenVox
